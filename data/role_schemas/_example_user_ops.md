# Role Schema: User Operations

## Role Overview

User Operations manages the full lifecycle of users on a platform — from acquisition and onboarding to engagement, retention, and win-back. The role is data-informed but fundamentally human-centric: understanding why users behave as they do, and designing interventions that change that behavior at scale.

---

## Sub-Roles

| Sub-Role | Focus | Common Employer Type |
|----------|-------|---------------------|
| User Lifecycle Ops | Onboarding, activation, retention campaigns | Consumer app, SaaS |
| Community Ops | UGC moderation, creator relations, forum management | Social platform, content app |
| Activity / Event Ops | Campaign design, seasonal promotions, gamification | E-commerce, gaming, app |
| VIP / Member Ops | Loyalty programs, high-value user management | Retail, fintech, platform |
| Content Ops | Content curation, creator incentive programs | Video platform, knowledge platform |

---

## Responsibilities

- Analyze user funnel to identify activation bottlenecks and drop-off points
- Design and execute retention campaigns (push notifications, in-app messages, email sequences)
- Build and maintain user segmentation models (RFM, behavioral cohorts)
- Run A/B tests on onboarding flows, re-engagement messages, and incentive structures
- Track and report core platform health metrics on a weekly/monthly basis
- Coordinate with product, growth, and engineering on feature changes that affect user behavior
- Manage community health: content quality, violation handling, creator incentives
- Develop user research insights through surveys, interviews, or behavioral data analysis

---

## Core Metrics

| Metric | Definition | Why It Matters |
|--------|-----------|----------------|
| DAU / MAU | Daily / Monthly active users | Platform health and engagement scale |
| DAU/MAU Ratio | Stickiness indicator | How habit-forming the product is |
| Retention Rate | % users returning after Day 1 / 7 / 30 | Product stickiness by cohort |
| Churn Rate | % users who stop using the product | Inverse of retention; loss signal |
| Activation Rate | % new users completing a key action | Onboarding effectiveness |
| LTV | Lifetime value of a user | Revenue potential per user |
| NPS | Net Promoter Score | User satisfaction and word-of-mouth potential |
| Notification CTR | % users clicking push notifications | Message relevance and timing |
| Reactivation Rate | % churned users returning after campaign | Win-back effectiveness |
| Content Engagement | Views, likes, shares, comments per piece | Content quality signal |

## Required Tools

- **Data platforms**: SQL for behavioral queries, internal BI tools, Google Analytics / app analytics SDKs
- **CRM / messaging**: push notification platforms (e.g., getui, jpush, internal systems), email service providers
- **Survey tools**: Typeform, Wenjuanxing, or in-app feedback tools
- **Collaboration**: Jira/Linear for coordination with product and engineering
- **Excel/Sheets**: cohort analysis, user segment sizing, campaign result tracking

---

## Interview Focus

- **User insight questions**: "Walk me through how you would investigate a 10% drop in D7 retention"
- **Retention scenario**: "Design a 30-day onboarding campaign for a new user who just registered but has not completed a key action"
- **Cross-team collaboration**: "How would you work with the product team to implement a behavioral change?"
- **Metrics decomposition**: "DAU dropped 15% last Monday. How do you diagnose this?"
- **Prioritization**: "You have three retention initiatives. How do you decide which to run first?"

---

## Portfolio Suggestions

1. **Retention Analysis**: Use a public dataset (e.g., e-commerce transaction data on Kaggle) to calculate cohort retention, identify a drop-off pattern, and propose an intervention
2. **User Segmentation Exercise**: Segment a dataset using RFM analysis and describe what campaign you would run for each segment
3. **Campaign Design Document**: Write a full campaign brief for a re-engagement push notification series — include target segment, message copy, timing logic, and success metrics
4. **Onboarding Teardown**: Analyze the onboarding flow of an app you use, document where users likely drop off, and propose two improvements with expected impact
